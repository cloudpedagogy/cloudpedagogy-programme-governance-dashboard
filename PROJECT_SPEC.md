# PROJECT_SPEC: cloudpedagogy-programme-governance-dashboard

## 1. Repo Name
`cloudpedagogy-programme-governance-dashboard`

## 2. One-Sentence Purpose
A high-level oversight platform for auditing curriculum alignment, quality assurance, and AI-governance status across entire academic programmes.

## 3. Problem the App Solves
Institutional "Blindness" at the programme level; provides a unified dashboard for viewing module reuse, alignment evidence (Assessment-to-Outcome), and formal AI-usage audit records.

## 4. Primary User / Audience
Quality assurance officers, deans, programme leads, and external examiners.

## 5. Core Role in the CloudPedagogy Ecosystem
The "Governance Aggregator"; nests modules and outcomes inside a "Programme" wrapper and attaches institutional sign-off metadata and AI-governance records.

## 6. Main Entities / Data Structures
- **Programme**: The high-level container (Title, Description, Metadata).
- **Module**: Reusable educational units nested inside the Programme.
- **Outcome/Assessment/Skill**: Detailed curriculum nodes linked via IDs.
- **GovernanceMetadata**: Tracking status (Draft/Approved), versioning, and reviewer logs.
- **AIGovernanceRecord**: A specific audit record for any entity tracking AI-usage, human-review status, and rationale.
- **Snapshot**: Point-in-time metrics (e.g., Reuse rates) for institutional reporting.

## 7. Main User Workflows
1. **Programme Audit**: Navigate through programme-module hierarchies to verify structural integrity.
2. **Alignment Review**: Inspect the evidence linking assessments to outcomes.
3. **AI-Use Logging**: Record and review how AI was used in the design or delivery of specific entities.
4. **Snapshotting**: Capture and compare metrics over time to track institutional maturity.

## 8. Current Features
- Hierarchical (nested) curriculum browser.
- Multi-entity AI-governance recording system.
- Structural metric snapshots.
- Governance status (review/approval) lifecycles.
- In-browser JSON dataset management.

## 9. Stubbed / Partial / Incomplete Features
- Not explicitly defined; app appears to be a functionally complete governance front-end.

## 10. Import / Export and Storage Model
- **Storage**: Persistent local storage (`pgd_dataset` and `pgd_snapshots`).
- **Import/Export**: Robust JSON import/export support for dataset portability.

## 11. Relationship to Other CloudPedagogy Apps
Sits at the top of the curriculum stack; provides the broad institutional "Wrap" for the granular data generated by the `mapping-engine` and `simulation-tool`.

## 12. Potential Overlap or Duplication Risks
Functionally overlaps with the mapping engine; distinguished by its "Hierarchical Nesting" (versus normalized graph) and focus on "Institutional Sign-off."

## 13. Distinctive Value of This App
The only tool that explicitly couples "Curriculum Structure" with a formal "AI Governance Record" for every single entity (programme down to assessment).

## 14. Recommended Future Enhancements
(Inferred) Automated "Green-Amber-Red" risk scoring for programmes with missing alignment evidence; integration with external examiner portals.

## 15. Anything Unclear or Inferred from Repo Contents
"ProgrammeAlignmentEvidence" score is inferred from the calculated density of relationships between Outcome and Assessment IDs.
